# 🧪 多题目识别功能测试指南

## 测试准备

### 测试图片准备
找一张包含多个题目的图片，例如：
- 数学试卷（一页3-5题）
- 英语作业（一页多道选择题）
- 物理练习题

### 测试步骤

#### 1. 打开应用
访问: http://localhost:5173

#### 2. 点击"录入错题"

#### 3. 上传测试图片
选择一张包含多个题目的图片

#### 4. 框选题目区域
- 在图片上拖动鼠标，框选第一道题
- 继续框选其他题目
- 点击"完成"按钮

**预期结果**:
- 每个框选区域显示题号（1、2、3...）
- 底部显示"已框选 N 个题目区域"
- 框选区域有蓝色边框和半透明背景

#### 5. 处理第一题
- 系统自动跳转到第一题
- 点击"🔄 重新识别"
- 查看OCR识别结果
- 填写错误答案和正确答案

**预期结果**:
- 识别出框选区域的文字内容
- 不是模拟数据，是真实的OCR结果

#### 6. 测试题目导航
- 点击"下一题"按钮
- 或点击题号按钮（2、3、4...）
- 切换到其他题目

**预期结果**:
- 能够流畅切换题目
- 每个题目的答案独立保存
- 不会相互干扰

#### 7. 批量保存
- 对每道题进行AI分析
- 点击"✅ 保存到错题本"
- 系统询问是否继续下一题

**预期结果**:
- 能够连续处理多道题目
- 每道题都保存到错题本
- 处理完成后显示提示

## 🐛 可能遇到的问题

### 问题1: 框选区域不准确
**解决**: 尽量精确框选，避免包含多余内容

### 问题2: OCR识别结果不完整
**解决**: 扩大框选区域，确保完整题目在范围内

### 问题3: 切换题目后答案丢失
**解决**: 这是bug，需要检查状态保存逻辑

### 问题4: 题目导航不工作
**解决**: 刷新页面重新开始，或检查控制台错误
